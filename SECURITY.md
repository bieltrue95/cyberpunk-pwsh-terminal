# Security Policy

## Reporting a Vulnerability or Sensitive Data Exposure

Please report security issues privately. Include:

- Affected file/path and commit reference (if known)
- Reproduction steps
- Impact summary
- Any exposed credential type (token, API key, private key, certificate, etc.)

## Reporting Channel

Use GitHub private vulnerability reporting for this repository when available.
If unavailable, contact the maintainer through the repository owner profile and avoid posting sensitive details in public issues.

## Expected Response

- Initial acknowledgement within 48 hours.
- Triage and containment guidance as soon as feasible.
- Remediation plan and follow-up update after verification.
